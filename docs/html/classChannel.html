Detailed Description

Channels enable communication between players.

Handles add and removal of users (listeners), and channel-specific patterns for broadcasting. Also handles an ACL-type interface for a minimum broadcast. Contains the information used for each channel (ID, history, listeners, access, and pattern. The only concept that must be explained is pattern. The pattern is what defines how messages are shown to players. Below is a list of modifiers; I have only used lowercase, but capitalizing the letter of the modifier will capitalize the first letter of it's translated value. p: The player name. m: The message. n: The channel name.
